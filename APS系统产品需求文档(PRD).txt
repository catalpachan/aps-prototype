================================================================================
                        1. 文档概述
================================================================================

1.1 文档目的
--------------------------------------------------------------------------------
本文档旨在明确跨厂协同APS智能排程系统的完整产品需求，为系统的设计、
开发、测试和验收提供统一的标准和依据。文档涵盖业务需求、功能需求、
技术架构、数据流设计、开发规划和验收标准等内容。

1.2 项目背景
--------------------------------------------------------------------------------
当前制造业面临总装厂与配套厂之间信息壁垒严重、排程决策依赖人工经验、
异常响应滞后、跨厂协同效率低等痛点。本项目旨在构建一套自主可控、
基于云原生架构的跨厂协同高级计划排程（APS）系统，实现从订单接入、
排程决策到执行反馈的全链路闭环管理。

1.3 术语定义
--------------------------------------------------------------------------------
APS     - 高级计划排程系统 (Advanced Planning and Scheduling)
BOM     - 物料清单 (Bill of Materials)
MES     - 制造执行系统 (Manufacturing Execution System)
ERP     - 企业资源计划 (Enterprise Resource Planning)
WMS     - 仓库管理系统 (Warehouse Management System)
IoT     - 物联网 (Internet of Things)
Agent   - 智能体/代理程序
LLM     - 大语言模型 (Large Language Model)
OEE     - 设备综合效率 (Overall Equipment Effectiveness)
UPH     - 单位小时产量 (Units Per Hour)
ASAP    - 尽快完成 (As Soon As Possible)
JIT     - 准时制生产 (Just In Time)

================================================================================
                        2. 产品概述
================================================================================

2.1 产品定位
--------------------------------------------------------------------------------
本产品是一套面向空调制造业的跨厂协同智能排程系统，服务于总装厂与
注塑/钣金/两器等配套厂之间的生产计划协同，通过数字化、智能化手段
实现生产资源的优化配置和生产效率的全面提升。

2.2 建设目标
--------------------------------------------------------------------------------
【核心目标】
- 打破总装厂与配套厂之间的信息壁垒
- 实现"需求、资源、规则、协同、监控"五大中心的深度协同
- 解决"生产什么、用什么生产、怎么生产合规、何时生产、生产得怎么样"
  五大核心业务问题

【具体指标】
- 订单准时交付率提升15%以上
- 设备综合利用率(OEE)提升10%以上
- 跨厂协同响应时间缩短50%以上
- 异常响应时间从小时级降至分钟级

2.3 设计理念
--------------------------------------------------------------------------------
【四大设计原则】
1. 功能模块化 - 各功能中心独立封装，便于扩展和维护
2. 技术微服务化 - 采用微服务架构，支持弹性伸缩
3. 数据一体化 - 统一数据标准，实现跨厂数据互通
4. 决策智能化 - 引入AI算法和智能体技术，实现智能决策

【业务闭环】
需求接入 → 资源匹配 → 规则校验 → 协同排程 → 监控反馈 → 优化迭代

【跨厂协同】
通过加密网关与统一数据标准，实现总装厂与配套厂在库存、产能、计划
层面的实时联动。

【智能驱动】
引入多智能体(Agent)协商机制与LLM助手，实现排程的自愈与优化。

2.4 核心价值
--------------------------------------------------------------------------------
1. 降本增效 - 优化排程方案，降低生产成本，提升设备利用率
2. 准时交付 - 精准排程计算，提升订单准时交付率
3. 异常自愈 - 智能诊断异常原因，自动触发重排
4. 跨厂协同 - 打破信息孤岛，实现产业链协同
5. 数据驱动 - 基于实时数据决策，持续优化改进

- 负责多源订单清洗、BOM智能递归拆解

- 通过数字孪生技术对设备、模具、人员、日历进行全要素建模

【协同中心】决策大脑
- 利用有限能力排程算法与多智能体协商机制
- 生成最优排程方案，处理异常自愈
- 核心功能：排程引擎、Agent协商、甘特图、诊断自愈、多方案对比

【监控中心】反馈优化闭环
- 聚合执行数据，监控"计划vs执行"偏差
- 计算KPI并反哺规则参数
- 核心功能：进度看板、异常哨兵、KPI分析、数据反哺、移动端

3.4 数据架构
--------------------------------------------------------------------------------

【核心数据模型】

1. 销售订单
   - 订单编号、客户信息、交期、批量、优先级
   - 旺季标识、清洗状态、工艺要求

2. 生产工单
   - 工单编号、父工单ID(跨厂关联)
   - 能效批次码(强一致性绑定)
   - 工序清单、物料需求、计划时间

3. 资源台账
   - 设备：编码、型号、状态、IoT标识、资质需求
   - 模具：编码、寿命、库位、跨厂共享标识
   - 人员：工号、技能矩阵、资质证书

4. 排程结果
   - 方案类型、甘特图数据、工序计划
   - 冲突点JSON、KPI快照

【数据流转逻辑(PDCA闭环)】

Inbound(入站):  ERP订单 → 需求中心(清洗/拆解/批次生成) → 生产工单库
Internal(内部): 工单需求 + 资源状态 + 规则参数 → 协同中心(Agent协商) → 排程结果
Outbound(出站): 排程结果 → MES/现场执行
Feedback(反馈): MES/IoT报工 → 监控中心(偏差分析) → 规则反哺/自愈重排
Cross-Plant(跨厂): 总装APS ↔ 加密网关 ↔ 配套APS(双向同步)

                        4. 功能需求
================================================================================

4.1 需求中心
================================================================================
【职能定位】订单流入口与预处理，实现总装订单到配套厂需求的精准拆分

4.1.1 订单同步与清洗
--------------------------------------------------------------------------------
【功能描述】
实时对接ERP销售订单、经销商订单系统，过滤无效订单，识别旺季订单特征

【输入】
- ERP销售订单数据
- 经销商订单系统数据
- 清洗规则配置

【处理逻辑】
1. 多源异构数据接入
2. 基于规则链过滤无效订单(冻结/草稿/数量不足)
3. 自动识别"旺季"特征并打标

【输出】
- 有效订单清单
- 旺季订单标识(红色边框高亮)
- 产能预警通知

【前端交互】
- 订单列表页：状态筛选、多维度排序
- 旺季订单红色高亮显示
- 可拖拽的清洗规则配置器

4.1.2 智能BOM拆解
--------------------------------------------------------------------------------
【功能描述】
总装订单自动拆解为注塑、钣金、两器等配套件需求，建立父子工单关联

【输入】
- 总装订单数据
- BOM基础数据
- 能效批次码要求

【处理逻辑】
1. 递归拆解总装父订单
2. 生成跨厂子工单(注塑/钣金/两器)
3. 生成全局唯一"能效批次码"并强制绑定

【输出】
- 父子工单关联表
- 跨厂需求清单
- 绑定批次码的工单树

【前端交互】
- 父子工单关联树状图
- 点击子工单双向溯源父订单及批次码
- 批次码颜色标注

4.1.3 库存可用性管理
--------------------------------------------------------------------------------
【功能描述】
实时同步库存状态，动态调整安全库存阈值，支持替代料匹配

【输入】
- 总装厂成品/半成品库存
- 配套厂关键零部件库存
- 替代物料匹配规则

【处理逻辑】
1. 动态计算各工厂安全库存水位
2. 基于"库存量优先"或"匹配度优先"推荐替代料
3. 库存低于阈值时同步预警

【输出】
- 库存可用性报告
- 替代料推荐清单
- 短缺预警信息

【前端交互】
- 跨厂库存动态监控看板
- 水位线拖拽调整
- 物料替代弹窗

4.1.4 插单池与优先级定义
--------------------------------------------------------------------------------
【功能描述】
建立优先级梯度，插单时自动评估配套厂产能占用和物料齐套性

【优先级梯度】
战略客户 > 旺季热销 > 常规订单

【输入】
- 插单需求数据
- 订单优先级规则
- 配套厂产能数据

【处理逻辑】
1. 评估配套厂产能占用情况
2. 评估物料齐套性
3. 独立沙箱模拟插单影响

【输出】
- 插单影响报告
- 缺料清单
- 预计延误天数
- 插单排程预方案

【前端交互】
- 插单影响分析卡片
- 缺料清单展示
- 延误天数预估

4.1.5 订单变更/撤销管理
--------------------------------------------------------------------------------
【功能描述】
支持订单交期调整、数量增减、紧急撤销，自动计算变更影响范围

【输入】
- 订单变更指令(交期/数量/撤销)
- 当前排程计划

【处理逻辑】
1. 自动计算变更对配套厂的影响范围
2. 建立变更闭环确认流程
3. 同步变更至关联工单

【输出】
- 订单变更影响报告
- 协同确认结果
- 重排建议

4.2 资源中心
================================================================================
【职能定位】生产要素全生命周期管理，提供跨厂资源负荷预判与共享调度

4.2.1 全要素设备建模
--------------------------------------------------------------------------------
【功能描述】
维护总装线、配套厂设备基础参数，配置IoT实时状态交互

【输入】
- 设备基础参数(型号、产能、维保周期)
- IoT接入配置
- 资质绑定信息

【处理逻辑】
1. 建立设备数字孪生模型
2. 接入IoT实时流(运行/故障)
3. 标记持证操作属性

【输出】
- 设备可视化档案
- 实时状态数据
- 异常预警

【前端交互】
- 三维设备卡片墙
- 实时状态徽标(绿/黄/红)
- 点击钻取查看实时参数

4.2.2 模具与工装档案
--------------------------------------------------------------------------------
【功能描述】
记录模具全生命周期，追踪跨厂调拨轨迹

【输入】
- 模具基础数据(一模几穴、适配机台、寿命)
- 库位状态
- 跨厂调拨物流时效

【处理逻辑】
1. 建立模具全生命周期档案
2. 剩余寿命自动扣减
3. 追踪跨厂调拨轨迹

【输出】
- 模具工装台账
- 寿命预警通知
- 调拨轨迹记录

【前端交互】
- 模具寿命进度条
- 预警红灯提示
- 调拨轨迹地图

4.2.3 人员技能矩阵
--------------------------------------------------------------------------------
【功能描述】
建立"工号-技能-等级"模型，支持跨厂支援人员管理

【输入】
- 工号、技能类型、技能等级
- 资质证书
- 跨厂支援标识

【处理逻辑】
1. 建立人员技能矩阵
2. 定义熟练度对UPH的修正系数
3. 特殊工序资质强校验

【输出】
- 人员技能矩阵表
- 资质合规性校验结果
- 跨厂支援人员台账

【前端交互】
- 人员技能热力图
- 跨厂支援标签(显示1.5倍工时)
- 资质到期预警

4.2.4 多层级日历体系
--------------------------------------------------------------------------------
【功能描述】
管理工厂假日、车间排班、设备维保窗口，自动检测跨厂时间窗偏差

【输入】
- 工厂假日安排
- 车间排班计划(1/2/3班)
- 设备维保窗口
- 旺季加班需求

【处理逻辑】
1. 合并工厂日历与设备排班
2. 配置旺季加班日历(6-8月每周加2天)
3. 自动检测跨厂时间窗偏差(>2小时自动标记)

【输出】
- 标准化生产日历
- 时间窗偏差报告
- 对齐建议

【前端交互】
- 跨厂同步日历视图
- 红色高亮偏差时段
- 对齐建议弹窗

4.2.5 实时状态反馈
--------------------------------------------------------------------------------
【功能描述】
通过IoT采集设备运行状态，配套厂物料齐套状态

【输入】
- IoT设备采集数据(运行/故障/报修)
- 配套厂物料齐套数据
- 物流运输数据

【处理逻辑】
1. 实时汇总生产要素状态
2. 状态数据供排程引擎调用
3. 异常状态实时推送

【输出】
- 资源实时状态看板

4.3 规则中心
================================================================================
【职能定位】将空调行业工艺约束、跨厂协同要求转化为算法语言

4.3.1 硬约束规则库
--------------------------------------------------------------------------------
【功能描述】
固化空调行业工艺约束、交付约束、批次约束、安全约束

【约束类型】
1. 工艺约束
   - 两器铜管材质/管径匹配机型
   - 冷媒加注需在检漏合格后执行

2. 交付约束
   - 配套件交付时间早于总装工序开工时间≥2小时

3. 批次约束
   - 禁止不同批次配套件混线装配
   - 同批次空调使用同批次配套件

4. 安全约束
   - 危化品工序需符合安监规范

【输出】
- 标准化硬约束规则集
- 跨厂约束一致性校验报告

【功能描述】

【评分指标及权重】
- 交付准时率(40%)
- 设备利用率(30%)
- 换产成本(20%)
- 物料齐套率(10%)

【换产优化规则】
- 同系列机型优先排程
- 颜色换产"浅→深"

【输出】
- 软约束评分矩阵
- 换产优化建议方案

4.3.3 提前期参数池
--------------------------------------------------------------------------------
【功能描述】

【工序提前期】
- 注塑: 标准X小时，旺季压缩至0.8X
- 钣金: 标准Y小时，旺季压缩至0.8Y
- 两器: 标准Z小时，旺季压缩至0.8Z

【物流提前期】
- 短途: A小时
- 长途: B小时
- 旺季加急: 压缩20%

【输出】
- 动态调整的提前期参数表
- 旺季提前期压缩方案

4.3.4 临时复杂规则集
--------------------------------------------------------------------------------
【功能描述】
支持"锁定、封锁、成组"等复杂逻辑

【规则类型】
1. 锁定规则 - 锁定关键设备/工单
2. 封锁规则 - 封锁特定时间段
3. 成组规则 - 同批次/同类型工单成组排程

5.2 外部系统交互
================================================================================

【外部系统清单】
- MES系统 - 获取报工数据，下发排程计划
- IoT平台 - 获取设备实时数据
- ERP系统 - 获取订单/物料主数据
- WMS系统 - 获取库存数据
- 配套厂APS - 跨厂协同核心
- OA系统 - 推送告警/通知

【核心交互规范】

1. APS ↔ MES
   - MES→APS: 报工数据、工序异常、产能占用(RESTful API, JSON)
   - APS→MES: 工序排程计划、异常处理指令、产能锁定(RESTful API, JSON)

2. APS ↔ IoT
   - IoT→APS: 设备运行数据、模具状态(MQTT, 延迟≤500ms)
   - APS→IoT: 维保指令、模具调拨指令(MQTT, JSON)

3. APS ↔ ERP/WMS
   - ERP→APS: 订单基础数据、物料主数据(每日批量+实时同步)
   - WMS→APS: 实时库存数据、出入库记录(每15分钟批量)

4. APS ↔ 配套厂APS
   - 总装→配套: 配套需求计划、产能锁定指令(加密传输)
   - 配套→总装: 产能数据、交付进度、异常信息(每日3次+实时)

5. APS ↔ OA
   - APS→OA: 异常告警、验收通知、培训信息
   - OA→APS: 审批结果、培训签到

A. 核心业务流程图
--------------------------------------------------------------------------------

【订单到排程流程】
ERP订单 → 订单清洗 → BOM拆解 → 库存校验 → 资源查询 → 规则校验 → 智能排程 → 甘特图确认 → 下发MES

【异常处理流程】
异常检测 → 分级告警 → 根因诊断 → 自愈策略 → 局部重排 → 影响评估 → 计划更新 → 通知相关方

【KPI闭环流程】
MES报工 → 数据聚合 → KPI计算 → 偏差分析 → 参数反哺 → 规则更新 → 排程优化

B. 跨厂协同要点
--------------------------------------------------------------------------------
1. 所有跨厂模块测试需总装+配套厂双方参与
2. 测试报告/验收报告需双方签字/盖章
3. 跨厂数据传输需双重加密
4. 产能锁定/封锁需双方确认
5. 异常处理结果需同步反馈

C. 风险与应对
--------------------------------------------------------------------------------
【技术风险】
- 排程算法性能不达标 → 提前进行算法原型验证
- 跨厂数据同步延迟 → 优化网络架构，增加缓存机制

【业务风险】
- 跨厂业务规则不一致 → 项目初期统一规则标准
- 用户接受度低 → 加强培训，提供操作手册

【管理风险】
- 需求变更频繁 → 建立变更控制流程
- 跨厂协调困难 → 定期召开协同会议
