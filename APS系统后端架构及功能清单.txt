APS 系统后端架构与功能清单
一、 总体后端架构设计
后端采用 云原生微服务架构，以支撑“需求、资源、规则、协同、监控”五大中心的解耦与高
效协同。系统强化了跨厂（总装厂 vs 配套厂）的数据互通与多智能体（Agent）博弈能力。
核心服务域规划
系统划分为 8 个核心服务域，通过 API Gateway 统一对外暴露接口：
1. 基础与权限服务：统一身份认证、多租户（跨厂）权限控制、审计日志。
2. 需求与订单服务 ：全链路订单管理、BOM 结构化拆解、库存策略计算。
3. 资源与物联网服务：设备/模具数字孪生、IoT 实时数据流处理、工厂日历管理。
4. 规则与算法服务 ：行业工艺约束库、参数配置池、模拟仿真沙箱。
5. 协同与调度服务 ：APS 核心排程引擎、多智能体协商、自愈重排逻辑。
6. 监控与告警服务 ：全链路数据聚合、KPI 实时计算、异常哨兵系统。
7. 移动端适配服务 ：轻量化接口、扫码解析、多媒体上传处理。
8. 通用支撑服务 ：文件对象存储（OSS）、消息通知中心、大模型（LLM）网关。
二、 详细后端功能清单
1. 需求中心后端
核心职责：实现从外部订单接收到生产工单生成的全流程自动化，重点解决“BOM拆解”与“库
存动态平衡”。
● 1.1 订单同步与清洗引擎
○ 多源异构接入：适配器模式接入 ERP、经销商系统及电商平台，支持增量/全量同步。
○ 智能清洗过滤器：基于规则链过滤无效订单（如：状态=冻结/草稿、数量 < 最小起订
量），自动标记清洗日志 。
○ 旺季特征识别：自动比对订单日期与“旺季时间窗”参数，打上“旺季/加急”标签，触
发产能预警逻辑 。
● 1.2 智能 BOM 拆解与批次服务
○ 递归拆解算法：输入总装父订单，基于标准库自动生成注塑（外壳）、钣金（底盘）、
两器（冷凝器）等跨厂子工单树 。
○ 全局能效批次码：生成全局唯一的“能效批次码”，强制写入父子工单关联字段，确保
全链路同批次配套，防止混批 。

○ 关联溯源服务：提供双向穿透接口，支持前端通过子工单 ID 反查父订单及完整配套树 。
● 1.3 库存与插单模拟
○ 动态水位计算：根据淡旺季生产计划，动态计算各工厂（总装/配套）的安全库存阈值 。
○ 插单影响沙箱：在独立沙箱中预演新订单插入，计算对配套厂产能占用及物料齐套性的
影响，生成包含“缺料清单、预计延误天数”的评估报告 。
○ 替代料智能匹配：主料库存不足时，按“库存量优先”或“匹配度优先”算法推荐替代
方案 。
2. 资源中心后端
核心职责：建立生产要素的数字孪生模型，确保跨厂资源的实时可见与可用性校验。
● 2.1 资源建模与生命周期
○ 设备数字孪生：维护设备静态参数、IoT 实时状态（运行/故障/待机）、以及“持证上
岗”资质要求 。
○ 模具寿命管理：监听报工数据自动扣减模具寿命；当（设计寿命 - 已用寿命）< 阈值时，
触发预警并推送到前端看板 。
○ 跨厂模具调拨：记录模具在总装厂与配套厂之间的流转轨迹、物流时效及接收状态 。
● 2.2 多层级日历计算引擎
○ 日历统筹逻辑：合并工厂日历、车间排班、设备维保计划，生成最终“可用时间窗” 。
○ 跨厂偏差检测：自动比对总装厂与配套厂的可用时间，若偏差 > 2小时，自动标记异常
并计算对齐建议 。
● 2.3 跨厂人员与外协
○ 支援工时计算：识别“跨厂支援”标签人员，按特定倍率（如 1.5倍）核算工时成本 。
○ 外协任务分发：当内部产能负载率 > 100% 时，自动拆分工序生成外协子订单，并同步
检验状态接口 。
3. 规则中心后端
核心职责：作为排程的“立法机构”，管理所有硬性约束与优化目标。
● 3.1 规则解析与执行
○ 硬约束校验器：代码化执行强制规则（如：批次必须一致、配套件需早于总装 2小时交
付、冷媒加注需持证） 。
○ 软约束评分引擎：对排程方案进行打分（如：颜色切换“浅→深”得高分，同系列机型
连排得高分） 。

○ 协同封锁逻辑：监听总装线维保事件，自动在配套厂对应资源的时间轴上插入“虚拟占
用块”，防止无效生产 。
● 3.2 参数池与仿真
○ 多场景参数配置：存储“淡季/旺季/应急”多套参数包（提前期、物流时间），支持一
键热切换 。
○ 仿真沙箱服务：深度克隆当前生产快照至临时库，支持“What-If”假设分析（如：假设
模具故障，计算影响） 。
○ 冲突消解建议：分析仿真结果，识别时间重叠点，生成具体的消解策略（如：建议加班、
建议外协） 。
4. 协同中心后端
核心职责：作为系统的“大脑”，执行智能排程与实时调度指挥。
● 4.1 智能排程核心
○ 多智能体协商 ：模拟“总装智能体”与“配套智能体”的握手协议。总装发布需求，配
套反馈产能，通过多次迭代达成共识 。
○ 有限能力排程算法：支持正向（ASAP）与逆向（JIT）排程，基于资源日历计算精确到
分钟的工序计划 。
○ 实时拖拽校验：响应前端甘特图的拖拽请求，实时运行规则引擎校验合法性，返回“通
过”或“违规原因（如违反批次约束）” 。
● 4.2 诊断与自愈
○ 一键归因算法：基于决策树分析延误根因（物料短缺、模具故障、物流延误），并输出
权重 。
○ 智能自愈执行：封装原子级恢复操作（如：切换备用机台、拆分工单到外协），支持秒
级自动重排 。
● 4.3 协作与智能辅助
○ 实时协作锁 ：管理多计划员并发操作，对当前编辑行/表进行锁定，并推送协作状态 。
○ LLM 智能助手网关：封装 Prompt Engineering，将系统排程数据转化为 Prompt 发送
给大模型，返回自然语言建议（如“为什么订单 A 延误了？”） 。
5. 监控中心后端
核心职责：构建全链路数据闭环，确保“计划=执行”。
● 5.1 全链路数据聚合
○ 报工流处理：消费 MES/IoT 报工消息，实时更新工单进度（百分比、预计完工时间） 。

○ 异常哨兵服务：轮询比对“计划 vs 实际”，触发三级告警（一级停线、二级延误、三级
偏差），并推送到对应人员 。
● 5.2 KPI 计算与反哺
○ 指标计算引擎：实时计算 OEE、准时交付率、配套响应周期等核心 KPI 。
○ 参数反哺接口：基于历史 KPI 偏差，自动生成规则参数调整值（如：建议将注塑提前期
+1小时），支持一键写入规则中心 。
● 5.3 报表与合规
○ 国产化合规生成：生成带水印、数据来源标识的 PDF/Excel 报表，符合数据安全规范 。
○ 看板配置存储：保存用户自定义的监控大屏布局配置 。
6. 移动端与通用服务
● 6.1 移动端适配
○ 极简报工接口：优化 API 响应速度，支持扫码枪/摄像头快速解析工单二维码 。
○ 多媒体上传：接收现场异常的语音描述或照片，转存至文件服务器并关联异常工单 。
● 6.2 通用支撑
○ 操作审计：记录所有关键操作（排程调整、规则修改）的用户、时间、IP 及变更前后的
值 。
○ 文件服务：统一管理报表导出文件及异常附件 。

四、 接口规划方案
接口设计遵循 RESTful 规范，部分实时交互采用 WebSocket。
1. 需求类 API
○ POST /api/demand/clean：触发订单清洗规则。
○ POST /api/demand/bom/dissolve：执行 BOM 递归拆解与批次码生成。
○ POST /api/demand/simulate/insert：插单影响模拟（返回缺料清单与延误天数）。
2. 资源类 API
○ GET /api/resource/mold/track：查询模具跨厂轨迹。
○ POST /api/resource/calendar/align：执行跨厂日历偏差自动对齐。
○ GET /api/resource/iot/stream：获取设备实时参数流（WebSocket）。
3. 协同排程 API
○ POST /api/schedule/drag/validate：甘特图拖拽实时校验。
○ POST /api/schedule/agent/negotiate：触发多智能体协商流程。
○ POST /api/schedule/heal：执行自愈策略（重排）。
○ POST /api/collaboration/lock：申请/释放协作锁。
4. 智能助手 API

○ POST /api/ai/ask：发送自然语言问题，返回排程建议（对接 LLM）。
5. 监控与移动端 API
○ POST /api/monitor/feedback/optimize：将 KPI 偏差转化为规则参数建议（反哺）。
○ POST /api/mobile/scan/parse：扫码解析与报工提交。
○ POST /api/mobile/upload/exception：上传异常图片/语音。
