APS 系统用户操作手册与交互原型设计
1. 系统概览
本系统采用“五大中心”架构，旨在解决总装厂与注塑/钣金/两器配套厂的跨厂协同排程问题 。
本手册指导用户如何通过可视化界面完成订单处理、资源管理、规则配置、协同排程及监控闭环。
2. 需求中心
2.1 订单接入与清洗
功能目标：识别有效订单，过滤无效需求，标记旺季加急单。
● 界面原型设计：
列表页布局：顶部为“清洗规则配置器”（含开关控件）。主体为表格，每行代表一个订单。
视觉高亮：旺季订单行背景色为浅红色，状态栏显示“🔥旺季”图标 。
过滤控件：左侧提供状态筛选器（冻结/草稿/有效）。
● 用户操作步骤：
1. 设置规则：点击配置器，拖拽滑块设置“最小起订量 > 50”，勾选过滤“草稿”状态 。
2. 筛选查看：在列表页筛选“有效”订单。
3. 查看详情：点击标红的订单行。
● 系统交互反馈：
○ 列表实时刷新，无效订单自动隐藏。
○ 系统Toast提示：“已自动识别15个旺季订单，触发产能预警” 。
2.2 智能BOM拆解与关联
功能目标：查看总装与配套厂的父子工单关系，确保能效批次一致。
● 界面原型设计：
分栏布局：左侧为“父子工单关联树”，右侧为详细属性面板。
树状图：根节点为总装订单，子节点为注塑/钣金工单。

色块标识：同一能效批次码（如Batch-A）的父子节点前显示相同颜色的色块（如绿色），
不同批次（Batch-B）显示不同颜色（如蓝色） 。
● 用户操作步骤：
1. 展开层级：点击树状图中的“总装父订单”的 + 号。
2. 点击子项：选中下方的“注塑外壳子工单”。
● 系统交互反馈：
○ 双向高亮：右侧面板立即高亮显示该子工单所属的父订单ID及绑定的“能效批次码”，
确认批次未混淆 。
2.3 插单模拟与库存预警
功能目标：模拟插单影响，处理物料短缺。
● 界面原型设计：
库存看板：仪表盘展示库存水位，包含一条可拖拽的虚线（安全库存线）。
弹窗：插单影响分析卡，包含雷达图（产能占用）和列表（缺料清单）。
● 用户操作步骤：
1. 调整阈值：在看板上按住安全库存线，向下拖拽调整阈值 。
2. 模拟插单：点击“插单模拟”，输入订单号和期望交期。
3. 处理缺料：当系统提示缺料时，点击“物料替代”按钮 。
● 系统交互反馈：
○ 水位变色：库存低于新阈值时，柱状图变为红色闪烁 。
○ 分析报告：弹出窗口显示“预计造成配套厂注塑机#3延误4小时，缺铜管200kg” 。
○ 替代推荐：列表展示“推荐替代料B（匹配度98%）”，点击确认后BOM自动更新 。
3. 资源中心
3.1 设备监控与资质校验
功能目标：查看设备状态及操作人员资质。
● 界面原型设计：
卡片墙：每个设备一张卡片，右上角有状态灯（绿=运行，红=故障）。
图标标识：需持证设备卡片上印有盾牌形“资质”图标 。
● 用户操作步骤：

1. 查看详情：点击“真空检漏机”卡片。
2. 分配人员：在弹出面板中，尝试指派一名无资质员工。
● 系统交互反馈：
○ 强阻断：系统弹出红色警告框“该员工无真空检漏资质”，并自动列出具备资质的人员
列表供选择 。
○ 实时数据：面板展开显示IoT采集的实时真空度数值 。
3.2 跨厂日历对齐
功能目标：发现并解决总装与配套厂的时间窗冲突。
● 界面原型设计：
双排甘特图：上方时间轴为总装厂，下方为配套厂。
冲突色块：两厂工作时间不重叠且偏差>2小时的区域，填充显眼红色背景 。
● 用户操作步骤：
1. 点击偏差：点击红色高亮的时间段（例如：总装周六加班，配套休息）。
2. 执行对齐：点击弹窗中的“一键同步”按钮 。
● 系统交互反馈：
○ 系统自动修改配套厂日历，红色区块消失，提示“日历已对齐，配套厂增加周六班次” 。
4. 规则中心
4.1 规则配置与权重调整
功能目标：调整排程算法的优先级和约束。
● 界面原型设计：
控制台：左侧为硬约束开关（Toggle Switch），右侧为软约束权重滑块（Slider）。
预览窗：底部悬浮小窗口，显示“当前规则下的预计交付率”。
● 用户操作步骤：
1. 开启约束：打开“批次混线禁止”开关 。
2. 调整权重：将“颜色换产优化（浅→深）”的权重滑块从30%拖至80% 。
● 系统交互反馈：
○ 动态图表：权重饼图实时变化。

○ 即时预览：底部预览窗提示“预计换产时间减少15%，但交付率可能下降1%” 。
4.2 协同模拟（What-If）
功能目标：模拟设备故障对交付的影响。
● 界面原型设计：
沙盘工作台：顶部为假设条件输入栏，中部为模拟甘特图。
对比面板：分左右两栏展示“基准方案”vs“模拟方案” 。
● 用户操作步骤：
1. 设定场景：点击“新增事件”，选择“注塑机故障”，输入时长“4小时” 。
2. 运行模拟：点击“开始推演”。
● 系统交互反馈：
○ 甘特图刷新，受影响的工单显示红色波浪线。
○ 系统生成报告：“交付准时率从99%降至95%，建议启用外协” 。
5. 协同中心
5.1 跨厂联动排程
功能目标：手动调整总装计划，自动联动配套厂。
● 界面原型设计：
联动甘特图：
○ 上层：总装生产线视图。
○ 下层：配套注塑/钣金线视图。 *
连线：点击工单时，显示父子工单间的虚线连接 。
● 用户操作步骤：
1. 拖拽操作：鼠标按住总装工单条，向后拖动2小时。
2. 松开鼠标：完成调整。
● 系统交互反馈：
○ 自动跟随：下层的配套件生产计划条自动向后平移，保持由于“提前期”设定的时间差 。
○ 冲突校验：若拖动位置导致违反“同批次”约束，工单条自动弹回原位，并提示“违反
能效批次约束” 。

5.2 智能诊断与自愈
功能目标：通过AI解决延误问题。
● 界面原型设计：
侧边栏助手：聊天对话框界面。
自愈卡片：系统主动推送的建议卡片，含“应用”和“忽略”按钮。
● 用户操作步骤：
1. 语音提问：点击麦克风，说“为什么这个订单延误了？” 。
2. 执行自愈：在助手回复的建议卡片中，点击“切换备用配套厂” 。
● 系统交互反馈：
○ 归因分析：助手回复“模具M-01故障导致（权重80%）”，并在甘特图上以热力图形
式高亮瓶颈工序 。
○ 秒级重排：点击应用后，进度条闪烁，甘特图瞬间刷新，显示切换工厂后的新排程 。
6. 监控中心
6.1 全链路进度与告警
功能目标：监控生产全过程，处理异常。
● 界面原型设计：
流程进度条：横向展示 配套生产 -> 物流 -> 总装 -> 检测。节点下方显示百分比 。
顶部通知条：不同颜色的横幅（红/黄/蓝）代表不同等级风险 。
● 用户操作步骤：
1. 查看告警：点击顶部的红色通知条“一级风险：注塑缺料停线” 。
2. 填写反馈：进入处理页，输入“已协调加急物流”，点击提交 。
● 系统交互反馈：
○ 系统记录闭环信息，告警条消失，异常状态更新为“处理中” 。
6.2 KPI反哺优化
功能目标：将执行数据转化为规则参数。
● 界面原型设计：
KPI看板：展示准时交付率趋势图。

反哺按钮：指标卡片旁的小闪电图标“⚡一键反哺” 。
● 用户操作步骤：
1. 点击反哺：发现近期物流常延误，点击“一键反哺”。
● 系统交互反馈：
○ 系统弹窗：“根据最近30天数据，建议将‘物流提前期’从2小时调整为3小时”。
○ 用户点击“确认写入”后，系统自动更新规则中心参数 。
7. 移动端应用
7.1 扫码报工与异常上报
功能目标：一线人员快速反馈。
● 界面原型设计：
极简首页：屏幕中央巨大的“扫一扫”按钮。下方为“我的任务”列表 。
异常入口：红色的“异常上报”悬浮球 。
● 用户操作步骤：
1. 扫码：用手机摄像头扫描设备上的二维码。
2. 上报异常：点击异常球，拍照上传现场模具损坏照片，语音输入“模具卡顿” 。
● 系统交互反馈：
○ 任务识别：扫描后立即显示当前工单任务，点击“开工”按钮变绿 。
○ 即时同步：PC端监控中心立即收到包含现场照片的异常告警 。
